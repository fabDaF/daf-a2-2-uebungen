## 4. Datei-Typen und ihre Tab-Struktur

Jede Lingoda-Lektion hat einen Buchstaben-Code. Dieser bestimmt den Tab-Aufbau der HTML-Datei.

### V — Vokabular (`*V*.html`)

**Zweck:** Neue Wörter mit Fotos, Kollokationen und Übungen einführen.

**Tab-Struktur:**
| Tab | Emoji | Name | Inhalt |
|-----|-------|------|--------|
| 0 | 📸 | Wörter | Foto-Vokabelkarten (10–12 Karten): Artikel · Wort · Plural · englische Übersetzung · **Beispielsatz aus Quasthoff-Kollokationen** |
| 1 | 💬 | Redemittel | Frage-und-Antwort Drag & Drop (Skill: daf-uebungsformen) |
| 2 | ✏️ | Lückentext | 2 Lückentexte in einem Tab, Live-Feedback, Timer |
| 3 | 📖 | Text & Fragen | Lesetext + Richtig/Falsch-Aufgaben |
| 4 | 🧩 | Satzbau | 6–8 Sätze, Drag & Drop (Skill: satzbau-drag-drop), Timer |
| 5 | 🔠 | Wortschatz | Alle Vokabeln: Artikel + Wort + Plural tippen, Live-Feedback, Timer |

**Referenzdatei (A2.1, genehmigt):** `DE_A2_1011V-meine-freunde.html`

---

### X — Extra-Übungen (`*X*.html`)

**Zweck:** Vertiefung und Anwendung des Vokabulars aus der V-Datei.

Erwartete Übungstypen: Zuordnung, Lückentext mit höherem Schwierigkeitsgrad,
Schreibaufgaben, situative Dialoge.

*(Tab-Struktur wird beim ersten A2.2-X-File definiert und hier ergänzt.)*

---

### G — Grammatik (`*G*.html`)

**Zweck:** Grammatikregel entdecken, verstehen und üben.

**Tab-Struktur:** *(Skill: daf-grammatik-uebungen — vor Erstellung lesen)*

Typische Tabs:
| Tab | Emoji | Name | Inhalt |
|-----|-------|------|--------|
| 0 | 🔍 | Entdecken | Beispielsätze, Regel induktiv ableiten (4 MC-Schritte + Regelkarte) |
| 1 | 📋 | Regel | Regelkarte, Paradigmentabelle |
| 2 | ✏️ | Lückentext | Grammatik-Lückentext (min. 10–12 Sätze), Live-Feedback, Timer |
| 3 | 🧩 | Satzbau | Sätze mit Grammatikfokus ordnen (6–8 Sätze), Timer |
| 4 | 🎯 | Multiple Choice | Richtig/Falsch oder A/B/C-Auswahl |

**⛔ Kein „Prüfen"-Button im Entdeckungs-Tab und in keinem anderen Tab.**

---

### R — Reading / Geschichte (`*R*.html`)

**Zweck:** Lesetext mit Vorentlastung, Verstehensaufgaben und Wortschatzübungen.

**Tab-Struktur:** *(Skill: a1-story-uebungen — vor Erstellung lesen und adaptieren)*

| Tab | Emoji | Name | Inhalt |
|-----|-------|------|--------|
| 0 | 🌅 | Vorentlastung | Schlüsselwörter, Bilder, englische Vorabinformation |
| 1 | 📖 | Geschichte | Originaltext |
| 2 | ❓ | Verständnis | Richtig/Falsch + offene Fragen |
| 3 | ✏️ | Lückentext | Sätze aus dem Text mit Lücken |
| 4 | 🧩 | Satzbau | Sätze aus dem Text zerlegt, Timer |
| 5 | 🔠 | Wortschatz | Schlüsselwörter tippen (Artikel + Wort + Plural), Timer |

---

### S — Speaking (`*S*.html`)

**Zweck:** Sprechanlässe, Dialoge, Redemittel für mündliche Produktion.

*(Tab-Struktur wird beim ersten A2.2-S-File definiert und hier ergänzt.)*

Erwartete Elemente: Dialogkarten, Sprechhilfen (Redemittel-Chips),
Rollenspiel-Szenarios, Satzanfänge zum Ergänzen.

---

### W — Wiederholung (`*W*.html`)

**Zweck:** Gesamtwiederholung einer Einheit oder des Kurses.

*(Tab-Struktur wird beim ersten A2.2-W-File definiert und hier ergänzt.)*

Erwartete Elemente: Gemischte Aufgaben aus allen Lektionen der Einheit,
Selbsteinschätzung, Lernziel-Checkliste.

## 5. Verbindliche Qualitätsregeln (für ALLE Dateitypen)

### Layout
- Lila Hintergrund-Gradient (`#667eea` → `#764ba2`) — Skill: `daf-html-layout`
- Weißer Container, max. 1000px, `border-radius: 12px`, `box-shadow: 0 10px 40px rgba(0,0,0,0.2)`
- Nav-Buttons: Emoji **über** Text (`nav-emoji` + `nav-label`, `flex-direction: column`)
- Aktiver Tab: Klasse `active` (nicht `aktiv`); sichtbare Section: Klasse `active`
- Mobile: `@media (max-width: 600px)` — `flex: 1 1 33%`, `flex-wrap: wrap`

### Lücken-Inputs — Placeholder IMMER leer

⛔ **`placeholder="___"` ist verboten.** Niemals Striche, Punkte oder andere Füllzeichen.

```html
<!-- ❌ FALSCH -->
<input placeholder="___">
<!-- ✅ RICHTIG -->
<input placeholder="">
```

Auch verboten: `placeholder="der/die/das"` — verrät den Anfangsbuchstaben.

Artikel-Input Placeholder: immer **`Artikel`** (das Wort, nicht `d__` oder `der/die/das`).

### Live-Feedback bei Texteingaben

Gilt für **alle** Tipp-Felder — Skill: `daf-uebungsformen` (Abschnitt „Live-Feedback")

| Zustand | Farbe |
|---------|-------|
| Leer | grau (neutral) |
| Prefix korrekt | lila (`:focus`) |
| Vollständig korrekt | grün (`.ok`) |
| Falsch | rot (`.no`) |

**Kein Prüfen-Button. Niemals.**

#### ⚠️ Kritischer Code-Fehler — absolut verboten

```javascript
// ❌ FALSCH — .ok wird NIEMALS gesetzt
if (ans.startsWith(val)) return;
inp.classList.add(val === ans ? 'ok' : 'no');

// ✅ RICHTIG — erst Gleichheit, dann Prefix
if (val === ans) { inp.classList.add('ok'); return; }
if (ans.startsWith(val)) return;
inp.classList.add('no');
```

### Wortschatz-Tab: makeWsInp()-Muster (verbindlich)

**Jeder Wortschatz-Tab verwendet drei separate Eingabefelder: Artikel + Wort + Plural.**

**Kein Komma zwischen Wort und Plural.** Kein Separator-Span.

```javascript
function makeWsInp(ans, placeholder, width) {
    var inp = document.createElement('input');
    inp.type = 'text'; inp.className = 'ws-inp';
    inp.placeholder = placeholder;
    inp.style.width = width + 'px';
    inp.dataset.ans = ans.toLowerCase();
    inp.setAttribute('autocomplete','off');
    inp.addEventListener('input', function() {
        timerAutoStart(5);   // Tab-Index anpassen
        var val = inp.value.trim().toLowerCase();
        var a = inp.dataset.ans;
        inp.classList.remove('ok','no');
        if (!val) return;
        if (val === a) { inp.classList.add('ok'); checkAllWs5(); return; }
        if (a.startsWith(val)) return;
        inp.classList.add('no');
    });
    return inp;
}
```

**WS_DATA Standardstruktur:**
```javascript
var WS_DATA = [
    { en: '(the smell)',     art: 'der', de: 'Geruch',      plural: '-\u0308e' },
    { en: '(to taste)',      art: '',    de: 'schmecken',    plural: '' },
    { en: '(the recipe)',    art: 'das', de: 'Rezept',       plural: '-e' },
];
```
- `art: ''` für Verben und Adjektive (kein Artikel)
- `plural: ''` wenn kein Plural existiert
- Plural-Notation: `-\u0308e` (Umlaut+e), `-\u0308er` (Umlaut+er) — nie `-äe` oder `-öer`

**CSS für Wortschatz-Inputs:**
```css
.ws-inp {
    border: none;
    border-bottom: 2px solid #c5cae9;
    background: transparent;
    outline: none;
    font-family: inherit;
    transition: border-color 0.2s;
}
.ws-inp:focus { border-bottom-color: #667eea; }
.ws-inp.ok { border-bottom-color: #27ae60; color: #27ae60; font-weight: 700; }
.ws-inp.no { border-bottom-color: #e74c3c; color: #e74c3c; }
.art { width: 70px; }
```

### Plural-Notation (verbindlich)

| Endung | Beispiel | Code |
|--------|----------|------|
| `-n` | die Frau → die Frauen | `"-n"` |
| `-en` | die Frage → die Fragen | `"-en"` |
| `-e` | der Tag → die Tage | `"-e"` |
| `-̈e` | der Ball → die Bälle | `"-\u0308e"` |
| `-er` | das Kind → die Kinder | `"-er"` |
| `-̈er` | das Buch → die Bücher | `"-\u0308er"` |
| `-s` | das Taxi → die Taxis | `"-s"` |
| `–` | der Lehrer → die Lehrer | `"–"` |
| `-̈` | der Vater → die Väter | `"-\u0308"` |

### Timer
- Tabs mit Tipp- oder Drag-Übungen bekommen einen Timer
- Timer-Zeile: `⏱ Zeit · 🏆 Bestzeit · ↺ Neustart · 💡 Lösungen` — alles in **einer Zeile** (`.btn-row`)
- Timer startet bei erster Interaktion (`timerAutoStart`), stoppt wenn alle korrekt (`timerStop`)
- Bestzeit wird in `localStorage` gespeichert (Key: `TIMER_PREFIX-best-N`)

### Satzbau
Verbindlich: Skill `satzbau-drag-drop` — `sbDragged`-Variable, `sb-bank-N`/`sb-row-N`/`sb-fb-N`-IDs, Fisher-Yates-Shuffle, 500ms-Timeout vor Farb-Feedback

### Bilder (Pexels) — Base64-Pflicht ab der ersten Zeile

**⚠️ PFLICHT: Base64-Einbettung direkt beim Erstellen**

Externe Pexels-URLs (`https://images.pexels.com/...`) sind **nicht zulässig**.
Alle Bilder müssen als Base64-Data-URLs direkt in die HTML-Datei eingebettet werden.

**Technische Realität (Stand 2026-03):**

Chrome (MCP) kann Pexels-Bilder laden, aber der MCP-Rückgabekanal blockiert Base64-Strings
(zu groß / Sicherheitsfilter). Der VM hat keinen direkten Internetzugang zu Pexels.

**Workflow für Bilder:**
1. Pexels-IDs per WebSearch suchen (`site:pexels.com/photo BESCHREIBUNG`)
2. HTML-Datei mit Pexels-URLs erstellen (sofort nutzbar online)
3. Frank führt lokal `embed_images.py` aus — einmalig, dauert ~30 Sekunden:
   ```bash
   cd ~/Documents/Claude/Cowork/htmlS/A2.2/
   python3 ../embed_images.py DE_A2_2011V-sinneswahrnehmungen.html
   ```
4. Danach ist die Datei vollständig offline-fähig mit eingebetteten Base64-Bildern

**URL-Schema für Pexels:**
```
https://images.pexels.com/photos/ID/pexels-photo-ID.jpeg?auto=compress&cs=tinysrgb&w=400&h=300&fit=crop
```

Die Dateien funktionieren auch **ohne** Base64-Einbettung vollständig — Bilder werden
dann live von Pexels geladen (erfordert Internetverbindung).

### Beispielsätze (nur V-Dateien)
Jeder Beispielsatz auf einer Vokabelkarte muss eine **belegte Kollokation** aus dem Quasthoff-Wörterbuch zeigen:
`Cowork/Grammatik/Worterbuch-Der-Kollokationen.txt`

Verben und Adjektive aus `V:AKK`, `V:SUBJ`, `A:`-Einträgen wählen.
Keine Allerweltsformulierungen wie „X ist schön" oder „Y ist wichtig".

### Copyright
Footer immer exakt so:
```html
<span class="author-name">&copy; Frank Burkert</span> &middot; DaF-Materialien &middot;
<a href="mailto:FrankBurkert@fabdaf.onmicrosoft.com">FrankBurkert@fabdaf.onmicrosoft.com</a>
```

⛔ **ABSOLUTES TABU — niemals ändern, niemals „verbessern", niemals durch eine andere Variante ersetzen.**
Der Footer ist das Aushängeschild des Autors. Keine Skill-Regel, kein Qualitätscheck und keine eigene Initiative rechtfertigt eine Änderung. Nur eine explizite schriftliche Anweisung des Benutzers darf den Footer-Text verändern.

## 6. Arbeitsreihenfolge beim Erstellen einer neuen Datei

1. **Lingoda-PDF lesen** (`pdftotext` → vollständigen Inhalt extrahieren)
2. **Passende Skills lesen** (je nach Datei-Typ)
3. **Bilder suchen und als Base64 einbetten** (WebSearch mit `site:pexels.com/photo` → sofort Base64)
4. **Kollokationen prüfen** (Quasthoff, nur für V-Dateien und G-Dateien)
5. **Datei aufbauen** — Tab für Tab, Skill-Muster einhalten
6. **Selbst-Check vor Auslieferung:**
   - [ ] Emojis über Text in Nav-Buttons?
   - [ ] Kein Prüfen-Button?
   - [ ] Live-Feedback korrekt implementiert (erst `val === ans`, dann `startsWith`)?
   - [ ] Timer-Zeile vollständig (Neustart + Lösungen in einer Linie)?
   - [ ] Satzbau nach Skill?
   - [ ] Wortschatz: 3 separate Felder (Artikel, Wort, Plural), KEIN Komma dazwischen?
   - [ ] Wortschatz: Artikel-Placeholder = `'Artikel'` (nicht `'der/die/das'`)?
   - [ ] Alle Bilder als Base64 (keine externen URLs)?
   - [ ] Copyright unverändert?
   - [ ] Beispielsätze aus Quasthoff belegt (V-Dateien)?
   - [ ] Plural-Notation korrekt (`-\u0308e` statt `-äe`)?
